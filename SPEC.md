# Визитка 2.0 — Спецификация
_Дата: 2026-04-21_

## Суть

Цифровая визитная карточка — PWA в нативной оболочке. Пользователь заполняет данные один раз, получает карточку с QR-кодом и кнопками контактов, делится ею вживую. Приложение для одного человека, хранение локально в `window.storage`.

## Цель улучшения

Текущая версия — MVP. Версия 2.0 добавляет:
- 4 шаблона дизайна (смена в одно касание)
- Соцсети: Telegram, Instagram, WhatsApp, LinkedIn
- Живой превью: карточка обновляется прямо в процессе редактирования
- Форм-фактор под стандарты рынка (QR как герой, кнопки-контакты, иконки соцсетей)

---

## Форм-фактор (по исследованию конкурентов)

Основано на анализе HiHello, Blinq, Taplink, Popl, Linktree.

### Главный экран (таб «Карточка»)

```
┌─────────────────────────────┐
│  [шаблон1] [шаблон2] [шаб3] │  ← горизонтальный скролл миниатюр
├─────────────────────────────┤
│         [фото]              │
│      Имя Фамилия            │  ← компактная карточка
│    Должность · Компания     │
├─────────────────────────────┤
│   [QR-код — большой, hero]  │  ← главный визуальный элемент
│   «Наведите камеру...»      │
├─────────────────────────────┤
│  [📞 Позвонить            ] │  ← полноширинные кнопки
│  [✉️ Написать на почту    ] │
│  [💾 Сохранить контакт    ] │
├─────────────────────────────┤
│  [tg] [wa] [ig] [li] ...   │  ← иконки соцсетей в ряд
└─────────────────────────────┘
```

### Экран редактирования (таб «Данные»)

- Форма с полями
- В шапке — миниатюра карточки (живой превью, ~120px высота)
- Карточка обновляется при каждом вводе (debounce 300ms)
- Кнопка «Сохранить» внизу

---

## Поля данных

### Личное
- Имя и фамилия (обязательное)
- Должность
- Компания
- Фото (загрузка + кроп, сохраняется base64)

### Контакты
- Телефон
- E-mail
- Сайт

### Соцсети (новое)
- Telegram (`@username` или `t.me/...`)
- WhatsApp (номер телефона)
- Instagram (`@username`)
- LinkedIn (`linkedin.com/in/...`)

### Дополнительно
- Адрес
- Заметка / описание

## UX-детали

### Выбор шаблона
- Горизонтальный скролл 4 миниатюр (80×50px) над карточкой
- Активный шаблон — золотая рамка / подчёркивание
- Тап — карточка перерисовывается с анимацией

### Живой превью в редакторе
- Мини-карточка (scale 0.55) в шапке экрана «Данные»
- Обновляется через debounce 300ms после ввода
- Показывает текущий выбранный шаблон

### Кнопки контактов на карточке
- Показываются только если поле заполнено
- «Позвонить» — `tel:` ссылка
- «Написать на почту» — `mailto:` ссылка
- «Сохранить контакт» — скачивание `.vcf` (логика уже есть)
- Иконки соцсетей — открывают ссылку в новой вкладке

### QR-код
- Кодирует vCard без фото (компактный, читается сканером)
- API: `qrserver.com` (уже используется)
- Размер: 220×220px
- Тап на QR — открывает модалку с увеличенным кодом (300×300px)

## Технические ограничения

- Один HTML-файл (`index.html`)
- Нет внешних JS-библиотек (CSS шрифты Google Fonts — допустимо)
- Хранение: `window.storage` (нативная оболочка)
- PWA: `manifest.json` + `sw.js` — остаются
- QR: внешний API `qrserver.com`

---

## Что НЕ входит в эту версию

- Аналитика (просмотры, сканирования)
- Несколько карточек на одном аккаунте
- Серверное хранение / облачная синхронизация
- NFC
- Drag-and-drop переупорядочивание полей
- Видео-интро

---

## Приёмочные критерии

1. ✅ Смена шаблона — тап, анимация, сохраняется при перезапуске
2. ✅ Заполненные соцсети появляются как иконки на карточке
3. ✅ Живой превью в редакторе обновляется без нажатия «Сохранить»
4. ✅ QR тапается — открывается увеличенный модал
5. ✅ «Сохранить контакт» скачивает `.vcf` с актуальными данными
6. ✅ Все 4 шаблона выглядят законченно на экранах 320–430px
7. ✅ Пустые поля не показывают пустые кнопки/строки на карточке
